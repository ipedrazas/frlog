# Journey 7: Guilt pile becomes visible and worth fixing

## Scenario
A manager repeatedly avoids a weekly admin cleanup task.

## Trigger
The user logs multiple low-energy notes over two weeks:
- “Still have not cleaned up task board”
- “Need to update roadmap statuses again”
- `.g overdue on weekly housekeeping`

## Flow
1. System clusters these into:
   - Guilt pile
   - Maintenance debt
2. Weekly review shows:
   “This is low urgency but consistently recurring.”
3. App suggests:
   - Watch
   - Worth fixing
4. User marks:
   - Worth fixing
5. App generates brief for:
   - recurring checklist
   - reporting assistant
   - automated reminder + template flow

### User goal
Understand whether avoided tasks are a workflow problem, not just personal failure.

### Product requirement
The app must frame guilt pile items compassionately.

### Success criteria
- user feels understood, not judged
- recurring maintenance work becomes visible
- output suggests practical interventions
