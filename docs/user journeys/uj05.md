# Journey 5: Weekly review leads to automation brief generation

## Scenario
A user has logged several frustrations about rewriting status updates across tools.

## Trigger
The weekly review shows “repeated status synthesis” as a top cluster.

## Flow
1. User opens the cluster.
2. App shows:
   - example notes
   - app context
   - estimated frequency
   - likely category: repetitive work / coordination tax
3. User clicks:
   - Generate automation brief
4. App creates a structured brief with:
   - problem
   - trigger
   - workflow
   - apps involved
   - examples
   - candidate solutions
   - agent-ready prompt
5. User clicks:
   - Copy as Agent Spec

### User goal
Go from annoyance to something actionable in one sitting.

### Product requirement
The generated brief must feel specific enough to be useful.

### Success criteria
- user copies/exports the brief
- brief includes examples and constraints
- user believes it is ready to hand off
