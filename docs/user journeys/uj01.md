# Journey 1: Instant friction capture during work

## Scenario
An engineer is preparing a status update and has to manually copy Jira issue IDs into Slack again.

## Trigger
The user feels a flash of annoyance:
“I have done this too many times.”

## Flow
1. User presses the global shortcut.
2. A Spotlight-style input appears.
3. User types:
   `Tired of copy-pasting Jira IDs into Slack`
4. User presses Enter.
5. Window disappears instantly.
6. Backend stores:
   - timestamp
   - note text
   - current app
   - recent app sequence
7. Later, system tags it as:
   - repetitive work
   - coordination tax
   - apps: Jira, Slack

### User goal
Capture the frustration with near-zero interruption.

### Product requirement
The flow must feel faster than deciding not to log it.

### Success criteria
- capture completed in under 5 seconds
- user returns to original work immediately
- metadata attached automatically
