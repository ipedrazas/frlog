# Journey 4: Context-switching pattern clarified by user feedback

## Scenario
A developer switches frequently between VS Code, browser docs, and Stack Overflow.

## Trigger
The app detects many app switches and raises the pattern in review.

## Flow
1. Weekly review shows:
   “High context-switching detected during debugging sessions.”
2. App asks:
   “Was this necessary research or workflow friction?”
3. User selects:
   - Necessary research
4. System stores feedback.
5. Similar future patterns are deprioritized unless accompanied by manual frustration logs.

### User goal
Avoid false positives and teach the system what kind of switching actually feels costly.

### Product requirement
The system must allow correction without argument.

### Success criteria
- user feels in control
- pattern quality improves over time
- app avoids shaming or overclassifying
