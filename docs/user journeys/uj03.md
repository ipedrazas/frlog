# Journey 3: Waiting-state logging reveals bottleneck

## Scenario
A designer is blocked waiting for stakeholder approval before continuing work.

## Trigger
The user realizes progress is stalled.

## Flow
1. User opens quick-capture.
2. Types:
   `.w waiting on homepage copy approval`
3. App stores waiting start.
4. Two hours later, user opens quick-capture again and types:
   `.w got approval`
5. App closes the waiting period and calculates duration.
6. During weekly review, the app shows:
   - total waiting time
   - recurring approval bottlenecks
   - affected tasks/projects

### User goal
Record blocked time without writing a detailed journal entry.

### Product requirement
The waiting-state interaction must be lightweight and forgiving.

### Success criteria
- user can start and stop waiting quickly
- waiting periods are summarized clearly
- blocked work becomes visible as a bottleneck category
