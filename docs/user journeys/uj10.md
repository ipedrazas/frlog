# Journey 10: Closed-loop outcome tracking

### Scenario
A user generated an automation brief and created a script from it.

### Trigger
A scheduled follow-up appears 10 days later.

### Flow
1. App asks:
   “Did the automation help with ‘Slack/Jira status updates’?”
2. User chooses:
   - Resolved
3. App asks optional follow-up:
   - “How much time do you think it saves per week?”
4. User enters:
   - “~45 minutes”
5. App updates the cluster as resolved and adds estimated recovered time to history.

### User goal
See whether the effort to fix something actually paid off.

### Product requirement
Friction Log should measure outcomes, not just observations.

### Success criteria
- user records a result
- app can report recovered value
- users see a narrative of improvement over time
