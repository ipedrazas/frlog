# Journey 9: User reviews evidence behind a cluster

## Scenario
A user sees a cluster labeled “Coordination Tax: Release approvals.”

## Trigger
The user wants to know whether the cluster is real or over-inferred.

## Flow
1. User opens cluster details.
2. App shows:
   - 5 manual notes
   - 3 waiting periods
   - app sequence patterns
   - confidence score
3. User sees the evidence and agrees.
4. User edits title to:
   - “Release approval chase”
5. User marks:
   - Worth fixing

### User goal
Trust the system because it is explainable.

### Product requirement
Every insight should be auditable.

### Success criteria
- user can inspect source examples
- confidence is visible
- user can correct naming and categorization
