# Journey 2: Passive pattern detection reveals hidden coordination tax

## Scenario
A PM feels busy all week but cannot explain where the time went.

## Trigger
The weekly review becomes available.

## Flow
1. User opens weekly review.
2. App shows:
   - 7 logs related to approvals and follow-ups
   - repeated app sequences across Slack, email, Notion, Jira
   - multiple waiting periods marked with `.w`
3. Summary reads:
   “You spent significant recurring effort chasing approvals for launch work.”
4. App presents:
   - top examples
   - estimated time cost
   - category: coordination tax
   - confidence: high
5. User marks cluster as:
   - Worth fixing

### User goal
Understand whether the feeling of busyness is tied to a repeatable problem.

### Product requirement
The app must turn scattered moments into a credible pattern.

### Success criteria
- user feels “yes, that is exactly what happened”
- summary is grounded in examples, not vague AI language
- user can prioritize the cluster
