# Jobs-to-be-Done

## Core functional job

When I feel recurring friction in my work,
I want to capture it quickly and see patterns over time,
so that I can decide what is worth automating, delegating, or redesigning.

## Core emotional job

When my work feels more annoying than it should,
I want reassurance that the problem is not just me,
so that I can turn vague frustration into a concrete improvement opportunity.

## Core outcome job

When I am ready to improve a workflow,
I want a structured brief I can hand to an AI agent or teammate,
so that I can move from noticing friction to actually reducing it.

---

## Primary JTBDs

### JTBD 1: Capture friction in the moment
When I notice a task is repetitive, annoying, or blocking progress,
I want to log it in a few seconds,
so that I do not lose the insight and do not interrupt my work.

### JTBD 2: Discover patterns I would not notice manually
When my week feels fragmented and reactive,
I want the system to detect recurring friction across apps, notes, and waiting periods,
so that I can identify what repeatedly drains time and attention.

### JTBD 3: Prioritize what is worth fixing
When I review my friction,
I want help distinguishing trivial annoyances from high-value automation opportunities,
so that I spend my effort on changes that matter.

### JTBD 4: Generate an automation-ready artifact
When I choose a friction pattern to address,
I want the app to turn it into a clear brief or agent spec,
so that I can immediately hand it off to an AI agent, script, or teammate.

### JTBD 5: Protect privacy while still getting value
When I install a desktop app that watches active windows,
I want strong control over what is collected,
so that I can trust the tool and use it during real work.

### JTBD 6: Close the loop on improvements
When I automate or redesign a workflow,
I want to mark whether it actually helped,
so that I can see whether Friction Log is producing real outcomes.

---

## Secondary JTBDs

### JTBD 7: Track coordination overhead
When I spend time chasing approvals, updates, or missing information,
I want to capture those delays and waiting periods,
so that I can identify the coordination tax in my work.

### JTBD 8: Track blocked work
When I am waiting on something or someone,
I want to mark that state quickly,
so that bottlenecks become visible without requiring a long explanation.

### JTBD 9: Track guilt-pile tasks
When there is work I keep postponing even though it matters,
I want to record those repeated avoidances,
so that I can see which obligations create recurring drag.

### JTBD 10: Separate necessary work from bad workflow design
When I switch tools frequently,
I want the app to help me distinguish valid research or creative work from avoidable context switching,
so that I do not over-automate the wrong things.


## Jobs-to-be-Done by Product Surface

## Quick Capture
Main job:
Help me preserve a moment of friction without disrupting work.

Measures of success:
- fast open
- fast submit
- no form fatigue
- instant close

## Passive Tracking
Main job:
Help me reconstruct enough context to make my logs useful later.

Measures of success:
- useful metadata
- low CPU
- trustworthy collection
- privacy controls

## Weekly Review
Main job:
Help me see what keeps happening and what matters.

Measures of success:
- signal over noise
- top 3 opportunities
- visible evidence
- clear prioritization

## Automation Brief
Main job:
Help me turn a frustration pattern into something actionable.

Measures of success:
- specific
- credible
- transferable
- editable
- useful to external AI tools

## Outcome Tracking
Main job:
Help me know whether a fix actually worked.

Measures of success:
- easy resolution updates
- visible recovered time
- product learns from outcomes
