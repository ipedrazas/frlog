
# User Personas and Their Main Jobs

## Persona A: Engineer
Context:
Works in VS Code, terminal, browser, Slack, Jira, docs, CI tools.

Main jobs:
- identify repetitive coordination and reporting work
- spot context-switching between coding and support/research tools
- generate briefs for internal tooling or scripts

Typical friction:
- copy-pasting IDs and logs
- repeated bug triage
- chasing approvals or clarifications
- waiting on CI, access, or review

## Persona B: Product / Ops Generalist
Context:
Works across Slack, Notion, docs, spreadsheets, tickets, email, calendar.

Main jobs:
- identify coordination tax
- identify recurring reporting/admin tasks
- generate workflow specs for automation tools or AI assistants

Typical friction:
- status update collection
- follow-up loops
- manually synthesizing information across tools
- recurring clean-up and admin

## Persona C: Founder / Manager
Context:
Works in communication-heavy environments across docs, meetings, approvals, planning tools.

Main jobs:
- identify bottlenecks and decision delays
- reduce communication overhead
- see what others wait on them for

Typical friction:
- repeated approvals
- context switching between strategic and operational work
- unresolved “guilt pile” admin
- manually preparing updates
